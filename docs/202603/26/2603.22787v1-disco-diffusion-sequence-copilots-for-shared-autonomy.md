## 摘要
共享自主结合了人类用户和 AI 副驾驶的动作，以控制诸如机械臂之类的复杂系统。当任务具有挑战性、需要高维控制或易受干扰时，共享自主可以通过使用经过训练的副驾驶，以与用户目标一致的方式有效纠正用户动作，从而显著提高任务性能。为了大幅提升共享自主的性能，我们引入了扩散序列副驾驶（DiSCo）：一种基于扩散策略的共享自主方法，能够规划与过去用户动作一致的动作序列。DiSCo 利用用户提供的动作对扩散过程进行种子引导（seeding）和修复（inpainting），并使用超参数来平衡对专家动作的遵循度、与用户意图的一致性以及感知的响应性。我们证明了 DiSCo 在模拟驾驶和机械臂任务中显著提高了任务性能。项目网站：https://sites.google.com/view/disco-shared-autonomy/

## Abstract
Shared autonomy combines human user and AI copilot actions to control complex systems such as robotic arms. When a task is challenging, requires high dimensional control, or is subject to corruption, shared autonomy can significantly increase task performance by using a trained copilot to effectively correct user actions in a manner consistent with the user's goals. To significantly improve the performance of shared autonomy, we introduce Diffusion Sequence Copilots (DiSCo): a method of shared autonomy with diffusion policy that plans action sequences consistent with past user actions. DiSCo seeds and inpaints the diffusion process with user-provided actions with hyperparameters to balance conformity to expert actions, alignment with user intent, and perceived responsiveness. We demonstrate that DiSCo substantially improves task performance in simulated driving and robotic arm tasks. Project website: https://sites.google.com/view/disco-shared-autonomy/

研究动机在于：现有的共享自主方法（如基于状态的扩散模型）通常只考虑单步动作纠正，缺乏时序一致性。论文提出了 **DiSCo (Diffusion Sequence Copilots)**，旨在利用扩散模型生成**动作序列**的能力，在纠正用户错误动作的同时，通过时序上下文确保 AI 的辅助与用户的意图高度对齐。

### 2. 核心方法论
DiSCo 的核心思想是将共享自主视为一个类似于图像修复（Inpainting）的过程，将用户的原始输入作为“粗糙草图”，通过扩散模型精炼为“专家级动作序列”。

*   **关键技术细节：**
    *   **种子引导 (Seeding)：** 将用户的动作序列 $U_t^0$ 进行 $k_{sw}$ 步前向扩散加噪，生成初始噪声序列 $U_t^{k_{sw}}$，以此作为反向扩散过程的起点。
    *   **动作修复 (Inpainting)：** 在反向扩散的每一步中，将当前生成的序列与用户原始动作序列进行融合。这确保了生成的未来动作不仅符合专家分布，还与用户过去的实际操作保持连贯。
    *   **可解释超参数：**
        *   **扩散比例 $\gamma$：** 控制对专家策略的遵循程度（纠错强度）。
        *   **修复比例 $\rho$：** 控制与用户意图的对齐程度。
        *   **混合比例 $\beta$：** 在最终输出时直接混合用户当前动作与 AI 预测动作，以提升实时响应性（Responsiveness）。
    *   **锚定策略 (Anchoring)：** 提出了“以用户为中心”和“混合锚定”两种方式来处理预测时界（Horizon）内的未来动作填充。

### 7. 优点与亮点
*   **序列化建模：** 突破了以往共享自主仅关注单步纠错的局限，利用扩散模型的序列生成能力提升了控制的平滑度和意图识别的准确性。
*   **超参数灵活性：** 提供的 $\gamma, \rho, \beta$ 三个参数为不同应用场景（如高性能要求的工业操作 vs 高自主权要求的辅助医疗）提供了精细调节手段。
*   **处理延迟：** 显式考虑了扩散模型的推理延迟，并通过退避时界（Receding Horizon）和动作混合技术解决了实时控制中的“卡顿”问题。
