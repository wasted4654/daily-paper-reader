# Set Diffusion: Interpolating Token Orderings Between Autoregression and Diffusion for Fast and Flexible Decoding 

Marianne Arriola 1 Volodymyr Kuleshov 1

## Abstract 

Discrete diffusion models have steadily improved in quality relative to autoregressive (AR) mod-els. However, these models are normally con-strained to fixed-length generation and do not sup-port key-value (KV) caching. Block diffusion partially bridges diffusion and AR by generat-ing token blocks left-to-right, but its fixed-size sequential blocks limit decoding flexibility and parallelism. Here, we present a new class of lan-guage models, set diffusion, comprised of (i) a likelihood parameterization that factorizes over flexible-position, flexible-length token sets and (ii) a set-causal diffusion architecture that sup-ports KV cache updates after every inference step. By factorizing over token sets instead of fixed-size blocks, tokens can be decoded in arbitrarily-ordered sets, including sliding-window sets, en-abling faster inference and support for any-order decoding. Set diffusion achieves better speed-quality tradeoffs on mathematical reasoning, sum-marization, and unconditional generation com-pared to prior diffusion language models while offering stronger infilling performance than block diffusion. However, they typically rely on architectures that preclude flexible-length generation (Kim et al., 2025a) and require full bidirectional context, which prevents key-value (KV) caching (Ma et al., 2025; Wu et al., 2025; Nguyen-Tri et al., 2025). Block diffusion (Arriola et al., 2025a) partially mitigates these challenges by modeling sequences autoregressively over contiguous left-to-right blocks with discrete diffusion applied within each block. This yields a tighter likelihood bound, variable-length generation, and KV caching for pre-viously decoded blocks. However, rigid left-to-right block structure restricts decoding flexibility for tasks requiring any-order generation such as infilling and arbitrary-position insertions (Li et al., 2025; Ma et al., 2026; Shu et al., 2026). Arriola et al. (2025a) also must delay KV cache updates until after a block is finished decoding because within-block denoising relies on bidirectional context (Wu et al., 2025; Seo et al., 2025). Finally, block diffusion limits parallel sam-pling since each block must be decoded before advancing (Liu et al., 2025b;a). Our key observation is that diffusion language models need not interpolate between autoregression and diffusion by varying a fixed block size; instead, they can interpolate by varying the order in which token sets are generated. Based on this, we introduce set diffusion, which defines an au-toregressive probability distribution over flexible-position, flexible-length sets of discrete random variables. Set diffu-sion generalizes block diffusion: fixed left-to-right blocks arise when sets are constrained to contiguous fixed-size blocks, while arbitrary-position and variable-length sets en-able flexible insertion, infilling, and sliding-window decod-ing. By interpolating between left-to-right and fully random orderings, set diffusion improves the tradeoff between sam-ple quality, inference efficiency, and decoding flexibility while supporting KV cache updates after every generation step. Across mathematical reasoning, summarization, and unconditional generation, set diffusion achieves better speed-quality tradeoffs than prior diffusion language models and substantially outperforms block diffusion on infilling. 1

## 3. Set Diffusion 

We introduce set diffusion models, which factorize gener-ation over flexible-position, flexible-length token sets. By varying the token-set partition and ordering distribution, set diffusion recovers autoregression, fully non-autoregressive diffusion (Sahoo et al., 2024; Ou et al., 2024; Shi et al., 2024), and block diffusion (Arriola et al., 2025a) as special cases. We present a likelihood parameterization that sup-ports variable-length sampling and a set-causal architecture that enables KV cache updates after every inference step.

## 4. Interpolating Between Token Orderings 

We now describe a structured family of ordering distribu-tions œÄ(œÉ) that interpolates between order-agnostic diffu-sion and autoregression by changing the order in which token positions are revealed. We specialize to fixed, data-and parameter-independent ordering distributions induced by position-dependent reveal-time schedules. This interpo-lation controls the speed‚Äìquality tradeoff in set diffusion: order-agnostic generation enables larger, more flexible to-ken sets for parallel decoding, while stronger left-to-right bias tightens the NELBO and improves sample quality. Block diffusion (Arriola et al., 2025a) instantiates this trade-off by varying a block size, but its fixed contiguous blocks restrict arbitrary-position insertion. Instead, we parameter-ize œÄ(œÉ) through position-dependent reveal-time distribu-tions: each token position is assigned a random reveal time, and sorting the sampled reveal times induces a generation order. The resulting position-offset schedules provide a simple mechanism for controlling left-to-right bias while retaining flexible-position decoding.

## 5. Sliding-Window Set Diffusion Language Models (SW-SetDLMs) 

We introduce Sliding-Window Set Diffusion Language Mod-els (SW-SetDLMs), a practical instantiation of set diffusion with a position-offset ordering distribution œÄ(œÉ), a factor-ized token-set likelihood, and an order-causal transformer architecture. These choices enable variable-length sliding-window decoding, KV cache updates after each inference step, and low-variance training. Unlike block diffusion (Ar-riola et al., 2025a), SW-SetDLMs decode flexible-position, flexible-length sets while preserving a left-to-right bias for high-quality generation.

5.2. Architecture 

SW-SetDLMs use a set-causal transformer whose attention pattern follows the sampled generation order. During train-ing, singleton token sets allow each input sequence to be permuted into generation order, reducing set-causal atten-tion to a reusable standard causal mask that avoids per-order mask construction and is compatible with compiled block-sparse attention implementations (Dong et al., 2024).

Comparison to Block Diffusion Block diffusion models (Arriola et al., 2025a) (also known as semi-autoregressive models (Han et al., 2022; 2023)) unmask tokens in strict left-to-right blocks, which sacrifices infilling flexibility and restricts parallelism, as each block must be fully decoded be-fore advancing. In contrast, we support decoding in sliding-window sets. While Arriola et al. (2025a) update KV caches once a block is fully decoded, we update the KV cache after every inference step for greater efficiency. 

Comparison to FlexMDM FlexMDM (Kim et al., 2025a) also uses any-order autoregression for flexible-length diffu-sion sampling. However, it introduces separate schedules for masked token insertion and unmasking, requires a separate insertion network to predict inserted tokens at each step, and uses a bidirectional transformer to parameterize token like-lihoods, precluding KV caching. In contrast, set diffusion explicitly models generation order, supports flexible-length generation without an insertion network while adhering to a user-specified inference budget, and enables KV caching. 

Comparison to Any-Order Autoregressive Models Our objective is related to the AO-ARM (Uria et al., 2014), whose objective is equivalent to fully non-AR diffusion (Hoogeboom et al., 2021; Ou et al., 2024; Uria et al., 2014; Shih et al., 2022). Unlike prior AO-ARMs which optimize over uniform orders and sample a single trajectory step dur-ing training (Uria et al., 2014; Hoogeboom et al., 2021; Ou et al., 2024), SW-SetDLMs use non-uniform orders and efficiently predict all token likelihoods simultaneously. While Shih et al. (2022) allow non-uniform œÄ(œÉ), their method cannot flexibly interpolate ordering distributions and uses a bidirectional transformer, preventing KV caching and variable-length inference. Methods that learn orderings via variational inference (Wang et al., 2025c) or confidence-based unmasking policies (Kim et al., 2026) require complex optimization to discover effective orderings, whereas we im-pose a structured prior over œÄ(œÉ) motivated by the success of block-wise generation (Arriola et al., 2025a). 

Limitations Although the gap between set diffusion and autoregression has narrowed, there is still a tradeoff between accuracy and parallelization in tuning the position-offset ordering schedules. The ideal schedule can be hand-tuned based on user preferences and/or parallelization capabilities of the inference hardware, or learned throughout training (Kim et al., 2026; Wang et al., 2025c). 

Conclusion This work introduces set diffusion, a frame-work for interpolating between AR and diffusion decoding by tuning generation orders rather than fixed block struc-ture. ## L. Experimental Details 

For GSM8K and CNN/DailyMail, we use AdamW with weight decay 1e‚àí5 and (Œ≤1, Œ≤ 2) = (0 .9, 0.98) . For OWT and LM1B, we use AdamW with (Œ≤1, Œ≤ 2) = (0 .9, 0.999) and no weight decay, following Sahoo et al. (2024); Arriola et al. (2025a). We apply gradient clipping with a maximum norm of 1.0. We maintain an exponential moving average (EMA) of model parameters with decay rate 0.9999, and use the EMA model for evaluation. For decoding, we use confidence-based greedy decoding (Nie et al., 2024). For parallel sampling, we use confidence-aware parallel decoding with tuned confidence thresholds (Wu et al., 2025). All reported decoding throughput measurements use inference batch size 1 on a single H100 80GB GPU. For unconditional generation using models trained on OWT and LM1B, throughput is instead measured on four 

RTX A6000 48GB GPUs. 

L.1. Mathematical Reasoning Data For GSM8K, we distill a student model initialized from Qwen/Qwen3-1.7B-Base to support parallel diffusion decoding, following the adaptation approach of Arriola et al. (2025b). Distillation mitigates catastrophic forgetting by training the student on responses generated by a stronger teacher model, aligning the adaptation objective with the teacher‚Äôs output distribution. Concretely, we construct the training set by prompting Qwen/Qwen3-32B-Base with each GSM8K training question and using the resulting responses as targets. We construct the validation set analogously by prompting 

Qwen/Qwen3-32B-Base with the GSM8K validation questions and using the resulting responses as validation targets. The prompts are drawn from the main configuration of the GSM8K dataset (Cobbe et al., 2021), downloaded from
