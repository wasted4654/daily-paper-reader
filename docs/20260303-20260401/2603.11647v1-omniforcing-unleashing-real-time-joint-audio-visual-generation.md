## 摘要
最近的联合视听扩散模型取得了显著的生成质量，但由于其双向注意力依赖性，面临高延迟问题，阻碍了实时应用。我们提出了 OmniForcing，这是第一个将离线双流双向扩散模型蒸馏为高保真流式自回归生成器的框架。然而，由于模态之间极端的时域不对称性以及由此产生的 token 稀疏性，直接对此类双流架构应用因果蒸馏会引发严重的训练不稳定性。我们通过引入一种带有零截断全局前缀（zero-truncation Global Prefix）的非对称块因果对齐（Asymmetric Block-Causal Alignment）来解决固有的信息密度差异，从而防止多模态同步漂移。因果偏移过程中由极端音频 token 稀疏性引起的梯度爆炸，通过配备恒等 RoPE 约束（Identity RoPE constraint）的音频 Sink Token 机制得到了进一步解决。最后，联合自强制蒸馏（Joint Self-Forcing Distillation）范式使模型能够在长序列生成过程中动态地自我纠正由曝光偏差（exposure bias）引起的累积跨模态误差。得益于模态独立的滚动 KV 缓存推理方案，OmniForcing 在单张 GPU 上实现了约 25 FPS 的最先进流式生成速度，同时保持了与双向教师模型相当的多模态同步和视觉质量。项目主页：https://omniforcing.com

## Abstract
Recent joint audio-visual diffusion models achieve remarkable generation quality but suffer from high latency due to their bidirectional attention dependencies, hindering real-time applications. We propose OmniForcing, the first framework to distill an offline, dual-stream bidirectional diffusion model into a high-fidelity streaming autoregressive generator. However, naively applying causal distillation to such dual-stream architectures triggers severe training instability, due to the extreme temporal asymmetry between modalities and the resulting token sparsity. We address the inherent information density gap by introducing an Asymmetric Block-Causal Alignment with a zero-truncation Global Prefix that prevents multi-modal synchronization drift. The gradient explosion caused by extreme audio token sparsity during the causal shift is further resolved through an Audio Sink Token mechanism equipped with an Identity RoPE constraint. Finally, a Joint Self-Forcing Distillation paradigm enables the model to dynamically self-correct cumulative cross-modal errors from exposure bias during long rollouts. Empowered by a modality-independent rolling KV-cache inference scheme, OmniForcing achieves state-of-the-art streaming generation at $\sim$25 FPS on a single GPU, maintaining multi-modal synchronization and visual quality on par with the bidirectional teacher.\textbf{Project Page:} \href{https://omniforcing.com}{https://omniforcing.com}

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：现有的联合音视频生成模型（如 LTX-2、Veo 3）虽然质量极高，但依赖**全序列双向注意力机制**。这意味着生成任何片段都需要处理整个时间线，导致极高的“首块输出延迟”（TTFC），无法满足实时交互或流式传输的需求。
*   **研究动机**：现有的解决方案要么是“先视频后音频”的级联模式（破坏了联合分布且难以流式化），要么是仅针对视频的自回归模型。直接将自回归蒸馏应用于音视频双流架构会因**模态间极端的时域不对称性**（如视频 3 FPS vs 音频 25 FPS）导致 token 稀疏，进而引发训练不稳定性（梯度爆炸）。
*   **整体含义**：OmniForcing 是首个将重型双向音视频基础模型蒸馏为高保真、实时流式自回归生成器的框架。

### 2. 方法论：核心思想与关键技术
OmniForcing 采用三阶段蒸馏流水线，将双向教师模型（LTX-2）转换为因果学生模型：
*   **非对称块因果对齐（Asymmetric Block-Causal Alignment）**：
    *   针对视频（3 FPS）和音频（25 FPS）的频率差异，建立以 **1 秒**为单位的“宏块（Macro-block）”对齐。
    *   引入**零截断全局前缀（Global Prefix）**：将 $t=0$ 时刻的初始帧设为全局可见，作为跨模态语义锚点，防止长序列生成的同步漂移。
*   **音频汇聚令牌（Audio Sink Token）与恒等 RoPE**：
    *   **挑战**：音频 token 极其稀疏，在因果掩码下，早期 token 的注意力分母极小，导致 Softmax 崩溃和梯度爆炸。
    *   **对策**：在音频序列前添加 16 个可学习的 Sink Tokens，并施加**恒等 RoPE（Identity RoPE）**约束，使其位置无关，充当全局内存缓冲区以稳定注意力熵。
*   **三阶段蒸馏流程**：
    *   **阶段 I（双向 DMD）**：利用分布匹配蒸馏（DMD）使模型具备少步（Few-step）去噪能力。
    *   **阶段 II（因果 ODE 回归）**：引入因果掩码，通过回归教师模型的 ODE 轨迹使权重适应因果生成。
    *   **阶段 III（联合自强力蒸馏）**：采用 **Self-Forcing** 策略，让模型在训练时基于自身生成的（而非真值）历史进行预测，从而纠正长序列生成的曝光偏差。
*   **推理优化**：利用音视频流在 Transformer 内部的解耦特性，实现**模态独立的滚动 KV 缓存**，将每步复杂度降至 $O(L)$。

### 6. 主要结论与发现
*   **实时性突破**：在单张 GPU 上实现了 **~25 FPS** 的生成速度，TTFC 仅为 **~0.7s**（相比教师模型的 197s 缩短了约 280 倍）。

*   **首创性**：填补了实时联合音视频流式生成的空白。
*   **架构优雅**：通过 1 秒宏块对齐自然地解决了音视频采样率不一致的问题。
*   **工程实用性**：提出的模态独立 KV 缓存和并行推理方案非常适合工业界部署。
