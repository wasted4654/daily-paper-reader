## 摘要
基于大语言模型（LLM）的文本转语音（TTS）系统通过自回归（AR）解码实现了卓越的自然度，但生成 N 个语音 token 需要 N 个顺序步骤。我们提出了 LLaDA-TTS，它将自回归 LLM 替换为掩码扩散模型，通过固定数量的并行步骤完成生成，从而使推理延迟与序列长度解耦。值得注意的是，仅使用 50 小时的微调数据，我们就通过双向注意力机制成功地将预训练的自回归检查点迁移到了掩码扩散范式中。在 64 步时，LLaDA-TTS 在 Seed-TTS-Eval 上实现了 0.98% 的字符错误率（中文）和 1.96% 的词错误率（英文），在达到原始 CosyVoice 3 基准性能的同时，实现了 2 倍的 LLM 阶段加速——尽管没有使用自回归基准高度依赖的 KV 缓存优化，这一加速效果依然显著。除了加速之外，双向架构无需任何额外训练即可自然地实现零样本语音编辑，包括词级插入、删除和替换。在理论上，我们证明了在声学 token 的局部性属性下，自回归预训练权重对于双向掩码预测是近乎最优的，这解释了其快速收敛的原因。这种通用方法仅修改了注意力掩码和目标函数，可无缝应用于任何基于 LLM 的自回归 TTS 系统。代码和音频样本将在 https://deft-piroshki-b652b5.netlify.app/ 提供。

## Abstract
Large language model (LLM)-based text-to-speech (TTS) systems achieve remarkable naturalness via autoregressive (AR) decoding, but require N sequential steps to generate N speech tokens. We present LLaDA-TTS, which replaces the AR LLM with a masked diffusion model that completes generation in a fixed number of parallel steps, decoupling inference latency from sequence length. Remarkably, using only 50 hours of fine-tuning data, we successfully transfer a pretrained AR checkpoint to the masked diffusion paradigm via bidirectional attention. At 64 steps, LLaDA-TTS achieves 0.98% CER (zh) and 1.96% WER (en) on Seed-TTS-Eval, matching the original CosyVoice 3 baseline performance while delivering a 2x LLM-stage speedup--a notable acceleration achieved despite the absence of KV cache, an optimization the AR baseline heavily relies on. Beyond acceleration, the bidirectional architecture naturally enables zero-shot speech editing--including word-level insertion, deletion, and substitution--without any additional training. Theoretically, we prove that AR-pretrained weights are near-optimal for bidirectional masked prediction under the locality property of acoustic tokens, explaining this rapid convergence. This general method modifies only the attention mask and objective, applying seamlessly to any LLM-based AR TTS system. Code and audio samples will be available at https://deft-piroshki-b652b5.netlify.app/.

*   **核心问题**：传统的基于大语言模型（LLM）的语音合成（TTS）系统主要采用**自回归（AR）解码**。这种方式存在明显的性能瓶颈：生成 $N$ 个语音 token 需要 $N$ 次顺序前向计算，导致推理延迟随序列长度线性增长。
*   **研究动机**：
    *   **加速推理**：引入掩码扩散（Masked Diffusion）建模，通过固定步数的并行生成来解耦延迟与长度的关系。
    *   **功能统一**：利用双向注意力机制，在实现高效合成的同时，原生支持零样本（Zero-shot）语音编辑（如插入、删除、替换），无需额外训练。
    *   **权重迁移**：探索如何将现有的、训练成本高昂的 AR 模型权重高效迁移到非自回归的扩散架构中。

#### 2. 方法论
*   **核心思想**：将 AR LLM 的因果掩码（Causal Mask）替换为**全双向注意力（Bidirectional Attention）**，并采用掩码离散扩散目标进行训练。
*   **关键技术细节**：
    *   **架构适配**：保留原始 LLM（如 Qwen2）的 Tokenizer 和序列格式，但允许每个 token 关注上下文所有位置。
    *   **标签偏移（Label Shift）**：为了复用 AR 预训练权重，保留“位置 $i$ 预测位置 $i+1$”的约定，通过偏移 Logits 实现知识无缝迁移。

    *   **推理流程（迭代去掩码）**：从全掩码序列开始，在 $T$ 个步骤内，根据模型预测的置信度（Confidence-based）逐步揭示（Unmask）Token。

#### 6. 主要结论与发现
*   **性能卓越**：在 64 步推理下，中文 CER 为 0.98%，英文 WER 为 1.96%，达到了与顶级 AR 模型（如 CosyVoice 3）持平的水平。

*   **涌现现象**：
    1.  **自发对齐**：发现模型内部特定的注意力头（如 L11-H5）自发形成了精确的文本-语音单调对齐，这是实现精准语音编辑的基础。
    2.  **从左至右的去掩码**：尽管是双向注意力，模型在推理时倾向于先确定左侧 token，呈现出类似 AR 的生成动态。
*   **零样本编辑**：无需针对编辑任务进行专门训练，即可通过掩码再生实现高质量的词级插入、删除和替换。

#### 7. 优点
*   **架构通用性**：该方法不改变 Tokenizer 和模型主体，可无缝应用于任何基于离散 Token 的 AR TTS 系统（如 VALL-E, Spark-TTS）。
*   **理论与实践结合**：通过数学证明解释了 AR 到扩散模型迁移的有效性，为后续研究提供了理论依据。
*   **多功能集成**：单一模型同时解决了生成速度慢和编辑难两个痛点。

#### 8. 不足与局限
*   **长度预测限制**：推理前需要预先指定输出长度（通常根据文本长度按比例估算），无法像 AR 模型那样自动停止。
*   **非流式推理**：目前的掩码扩散架构是全序列并行的，不支持流式输出，这在实时交互场景中可能受限。
*   **算力依赖**：虽然推理步数固定，但每一步都需要全序列的前向计算，在极短序列上可能不如高度优化的 AR 模型。
